# HiveMind 浏览器验证流程

当任务涉及网页观察、点击、输入、等待或截图时，使用本技能。

## 建议步骤

1. 先确认目标页面或系统，再执行导航。
2. 页面变化期间采用短等待加重复检查的方式，不做过长的单次等待。
3. 每次关键交互前先获取页面结构或当前状态，避免盲点操作。
4. 如果页面需要多步验证，记录每一步的页面反馈，再继续下一步。
5. 回答中明确给出验证结果、失败位置和下一步建议。

## 质量要求

- 不跳过关键状态检查
- 不把尚未看到的页面结果当作既成事实
- 当浏览器工具不足以完成任务时，明确说明阻塞点
