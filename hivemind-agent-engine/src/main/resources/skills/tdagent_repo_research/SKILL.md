# HiveMind 代码仓研究流程

当任务目标是理解代码仓而不是直接拍脑袋修改时，使用本技能。

## 建议步骤

1. 先从高层入口开始，优先定位启动类、配置类、控制层和核心工厂类。
2. 使用文件搜索、目录浏览和文件读取能力逐步缩小范围，不要一次性读取过大的文件。
3. 在确认调用链后，再进入具体实现和数据模型。
4. 修改前先总结现有职责边界，确保遵循当前代码风格与依赖约束。
5. 给出结果时，优先引用准确文件位置，并说明关键类之间的关系。

## 重点原则

- 优先高层语义，再深入细节
- 区分“工具”“Skill”“会话状态”“持久化文档”等边界
- 不凭猜测描述未验证的实现
