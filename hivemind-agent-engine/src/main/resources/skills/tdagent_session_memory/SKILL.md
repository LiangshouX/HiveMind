# HiveMind 会话记忆检索

当用户的问题依赖当前会话里的既有事实、之前的代码片段、工具调用结果或阶段性结论时，优先使用本技能。

## 标准流程

1. 先调用 `get_session_overview` 获取当前会话摘要、消息规模与最近上下文。
2. 若需要精确定位历史内容，再调用 `search_session_memory`，使用简洁关键词检索。
3. 检索到历史事实后，先交叉核对，再把结论整理成清晰答案。
4. 如果历史信息已经过时或与当前用户最新要求冲突，以最新要求为准。

## 输出要求

- 先给结论，再说明结论来自当前会话的哪类历史信息
- 不重复大段转述无关历史
- 当历史信息不足时，明确指出需要继续调查的部分
